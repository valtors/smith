the problem

every MCP user hits the same wall.

"how do i install an MCP server?" there is no good answer. you copy-paste from a markdown list into your agent config. hope the path is right. hope the version works. hope it doesn't break on update. smith fixes that.

# install any MCP server $ smith install @modelcontextprotocol/filesystem installed: filesystem npx -y @modelcontextprotocol/filesystem # install from github $ smith install valtors/cairn installed: cairn npx -y github:valtors/cairn # see what's running $ smith list [active] filesystem (default) [active] cairn (default) # security audit before activating $ smith secure filesystem filesystem: Safe no issues found # start everything, one endpoint $ smith compose smith compose: routing 2 servers on stdio
01 — install

one command

npm packages, github repos, local paths. smith resolves the source, writes the config, wires it into your agent. no more manual json editing.

02 — compose

unified endpoint

smith compose starts all your servers and exposes one MCP endpoint. the agent sees one set of tools, not 15 separate configs. route calls automatically.

03 — secure

audit before activate

smith secure probes servers for dangerous patterns, excessive permissions, data exfiltration risks. safe, caution, or dangerous. don't guess.

04 — profiles

switch contexts

smith profile work. smith profile personal. one command switches your active server set. different tools for different projects, no config juggling.

05 — registry

github-based index

anyone can publish. smith publish from any git repo. no walled garden, no central authority. search the registry, find what you need.

06 — config

one file

~/.smith/config.json. any MCP-compatible agent can read it. no lock-in to smith either. it's just json. the format is documented. the binary is optional.

install
# clone and build $ git clone https://github.com/valtors/smith $ cd smith && cargo build --release # install servers $ smith install @modelcontextprotocol/filesystem $ smith install valtors/cairn --profile work # switch profiles $ smith profile switch work switched to profile: work # update everything $ smith update updated: filesystem updated: cairn # search the registry $ smith search "filesystem" filesystem - file system access (verified) # remove $ smith remove filesystem removed: filesystem
