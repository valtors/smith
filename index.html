usage

one command beats ten steps

no sdk to learn, no config to write, no dashboard to configure. smith runs from your terminal.

$ smith install filesystem
  package parsed: filesystem@1.2.0
  config written: ~/.config/smith/servers.toml
  server verified: 12 tools available

$ smith install git memory
  installed: git (8 tools)
  installed: memory (3 tools)

$ smith compose fs git memory
  composing 3 servers into one endpoint
  merged: 23 tools on port 9847

$ smith scan
  filesystem  file read      safe
  memory      env access    warning
  shell exec  blocked

how it works

three steps, zero config

no sdk, no config files, no dashboard. just one binary and your terminal.

1

install

smith downloads and verifies the MCP server package. writes config automatically.

$ smith install filesystem
12 tools ready
2

compose

merge servers into one endpoint. automatic json-rpc multiplexing across all tools.

$ smith compose fs git
23 tools on :9847
3

scan

static analysis catches dangerous patterns before servers run. block by default.

$ smith scan
1 blocked, 1 warn
